Install PlanSync

Two commands and you're set:

$ npm install -g plansync
$ cd your-project
$ plansync init
  Press Enter → browser opens → authorize → done

Prerequisites

  • Node.js 18+ — any LTS version
  • A GitHub repo — with a remote named origin
  • Windows users: Git for Windows (includes Git Bash)

No API keys. No OAuth App registration. No environment variables. A built-in GitHub OAuth Client ID is included — device flow works out of the box.

How authentication works

When you run plansync init:

  1. Press Enter → PlanSync starts a 5-second countdown then opens your browser
  2. Visit github.com/login/device, enter the code shown in terminal, click Authorize
  3. PlanSync gets a token, saves it to .plansync/config.json, and adds it to .gitignore

Prefer a token? Paste a Personal Access Token with repo and project scopes at the prompt instead.

Next steps after init

Once plansync init completes, what you do next depends on whether you're starting fresh or adopting an existing project:

Starting a new project

  1. Create your repo on GitHub and git push to it
  2. Tell your agent to read AGENTS.md and write a plan to .plansync/plan.json
  3. Run plansync delegate — pushes Issues, board, scope files to GitHub, commits, and pushes
  4. Each collaborator runs plansync sync to lock scope + get per-user context files

Adopting an existing project

  1. Describe the next milestone to your agent — it will write a plan to .plansync/plan.json
  2. Run plansync delegate — same as new project: Issues, board, scope files
  3. Each collaborator runs plansync sync to lock scope + get per-user context files
  4. Change your mind? Run plansync clean to undo everything

See the How it works page for a full walkthrough with command examples.

Project owners: Use plansync sync --supervisor to keep all files writable while still generating context files. This lets you oversee all tasks without scope restrictions.

ErrorFix

Permission deniedRun sudo npm install -g plansync or fix npm permissions
